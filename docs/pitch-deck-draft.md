## 3｜工作流程：系統實際做什麼

每天只需執行一個命令，系統自動完成以下 5 個步驟：

**步驟一：選場景**
從場景庫中根據輪換規則選出今天的主題。同一個場景 14 天內不重複，同一類型不連續使用。

**步驟二：生圖**
用 AI 生成攝影風格的真實感照片。生成後自動做品質檢查（小吉清不清楚、體型反差夠不夠明顯、有沒有 AI 破圖）。不過關就重試一次。

**步驟三：寫文案**
用 AI 以小吉第一人稱寫 Instagram 文案。語氣：冷靜、直接、帶點理所當然的自信。加上 15–20 個 hashtag。

**步驟四：上傳排程**
圖片自動上傳到發文系統，準備排程。

**步驟五：你做決定**
看預覽，輸入一個字 `APPROVE`，明天早上 9:00 自動發出。

如果不滿意，可以說「改文案，把語氣改硬一點」或「圖片背景太模糊，修一下」，系統立刻重做，你再看一次。

## 3.5｜實際操作長什麼樣子（一次完整示範）

### 前置準備（每次開機後只需做一次）

打開終端機，輸入：
```
docker compose up -d
```
這個指令啟動後台服務（Postiz 跑在裡面，負責跟 Instagram 溝通）。
看到 5 個服務全部顯示「Started」就完成了，之後不需要再動它。

可以用瀏覽器開 `http://localhost:4007` 確認 Postiz 後台正常。

---

### 每天的日常操作

**第一步：打開終端機，進到專案資料夾**
```
cd ~/projects/xiaogi
```

**第二步：啟動 Claude（AI 助理）**
```
claude
```
等幾秒，看到輸入框出現就準備好了。

**第三步：輸入一個命令，啟動今天的內容生產**
```
/run
```

接下來什麼都不用做，大約等 1–2 分鐘。系統自動執行：
- 讀取小吉的規則書與歷史成效
- 選出今天的場景，寫好給 AI 的圖片指令
- 呼叫 Ideogram 生成圖片（這步最久，約 30–60 秒）
- 自動做品質檢查，必要時重試
- 寫文案、配 hashtag
- 上傳圖片到雲端（Cloudflare R2），取得公開連結
- 把所有東西送進 Postiz，準備排程

**第四步：終端機出現預覽，你做決定**

畫面會顯示像這樣：
```

==================================================

圖片位置：outputs/2026-04-01/image.png
預覽指令：open outputs/2026-04-01/image.png

── Instagram ──────────────────────────────────
我 8 點就到了。
文件昨晚看完了。
今天的會，我來主持。

你們慢慢來，我等。

#吉娃娃 #chihuahua #bossdog ...

預排時間：2026-04-02 09:00

輸入：APPROVE / REVISE CAPTION / REVISE IMAGE / SKIP
==================================================
```

**第五步：輸入 `APPROVE`**
系統建立排程，明天早上 9:00 自動發出，這裡結束。

## 5｜系統如何越跑越聰明

每篇貼文發布後 48 小時，系統自動讀取真實互動數據（按讚、收藏、分享），記錄下來。累積 5 筆後，它開始自動更新自己的「劇本」：

- 如果「主管類」場景持續高於平均 → 自動提高優先序
- 如果某種開場句式連續 3 次勝出 → 設為預設風格
- 如果某類場景持續沒表現 → 標記「避免使用」

**系統不需要人工調整，它從結果學習。**

## 5.5｜這個系統長什麼樣子：資料夾架構

就像一個小型出版社，每個部門各司其職：

**📁 knowledge（知識庫）— 人工維護，不常動**
- `character.md`：小吉長什麼樣、他的個性、絕對不能在圖片裡出現的東西
- `viral-formula.md`：所有場景分類與具體場景清單，加上輪換規則

**📁 skills（技能庫）— 系統從成效中學習，自動更新**
- `image-generation.md`：怎麼跟 AI 說才能生出好圖（prompt 公式與禁忌）
- `caption-writing.md`：怎麼寫出有感的文案（hook 類型、hashtag 策略）
- `performance-log.md`：每篇貼文的真實互動數據記錄

**📁 outputs（每日產出）— 系統自動建立，每天一個資料夾**
- 以日期命名，例如 `2026-04-01/`
- 裡面有：今天選的場景紀錄、給 AI 的圖片指令、生成的圖片、文案、發布結果

**📁 docker（後台基礎設施）— 只需設定一次**
- 管理 Postiz 和五個相關服務的設定
- 啟動後在背景默默執行，平時不需要干涉

**📁 docs（文件）**
- 專案進度快照（每次執行前系統自動讀取）
- 從零架設的 SOP、Docker 設定細節

**📁 references（小吉參考圖）**
- 小吉的標準外型照，每次呼叫 AI 生圖都會帶入
- 確保每張圖的小吉外型一致，不會每天長不一樣

```
xiaogi/
├── knowledge/
│   ├── character.md          ← 小吉的規則書
│   └── viral-formula.md      ← 場景庫
├── skills/
│   ├── image-generation.md   ← 生圖技能（會自動更新）
│   ├── caption-writing.md    ← 文案技能（會自動更新）
│   └── performance-log.md    ← 成效日誌
├── docker/
│   └── docker-compose.yml    ← 後台服務設定
├── docs/
│   ├── project-status.md     ← 目前進度快照
│   └── setup-sop.md          ← 架設 SOP
├── references/
│   └── 小吉.png              ← 角色參考主圖
└── outputs/
    └── 2026-04-01/
        ├── scene.md          ← 今天選的場景
        ├── prompt.md         ← 給 AI 的圖片指令
        ├── image.png         ← 生成的圖片
        ├── caption.md        ← Instagram 文案
        └── publish-result.md ← 發布結果
```
